# 分数换算关系说明

本文档说明系统中各种分数指标之间的换算关系。

## 📊 指标概览

系统使用以下指标来评估学生的数学能力：

1. **能力值（theta）**：IRT 模型的核心指标，范围 -3.0 到 3.0
2. **难度值（difficulty）**：题目的难度参数，范围 -3.0 到 3.0
3. **标准分数（Scaled Score）**：0-1000 分制，更直观的分数
4. **年级等值（Grade Equivalent, GE）**：表示相当于哪个年级的水平

## 🔄 换算关系

### 1. 能力值 → 标准分数

**公式**：`标准分数 = 500 + (能力值 × 100)`

| 能力值 | 标准分数 | 说明 |
|--------|---------|------|
| -3.0 | 200 | 最低水平 |
| -1.5 | 350 | 基础水平 |
| 0.0 | 500 | 中等水平（基准） |
| 1.5 | 650 | 良好水平 |
| 3.0 | 800 | 最高水平 |

**特点**：
- 线性映射，简单直观
- 500 分对应中等水平（能力值 0.0）
- 每 1 个能力单位 = 100 分

### 2. 能力值 → 年级等值

**分段线性映射**：

| 能力值范围 | 年级等值范围 | 对应学期 |
|-----------|-------------|---------|
| -3.0 ~ -1.5 | 6.5 ~ 7.0 | 低于七年级上 |
| -1.5 ~ -0.5 | 7.0 ~ 7.5 | 七年级上 |
| -0.5 ~ 0.0 | 7.5 ~ 8.0 | 七年级下 |
| 0.0 ~ 0.5 | 8.0 ~ 8.5 | 八年级上 |
| 0.5 ~ 1.0 | 8.5 ~ 9.0 | 八年级下 |
| 1.0 ~ 1.5 | 9.0 ~ 9.5 | 九年级上 |
| 1.5 ~ 3.0 | 9.5 ~ 10.0+ | 九年级下及以上 |

**示例**：
- 能力值 0.0 → 年级等值 8.0（八年级上水平）
- 能力值 1.0 → 年级等值 9.0（九年级上水平）
- 能力值 2.0 → 年级等值 9.8（接近十年级水平）

### 3. 能力值 → ZPD（最近发展区）

**公式**：
- ZPD 下界 = 能力值 - 0.5
- ZPD 上界 = 能力值 + 1.0

**含义**：
- 下界：稍低于当前水平，用于巩固基础
- 上界：挑战性题目，通过努力可以掌握

**示例**：
- 能力值 0.5 → ZPD 范围：0.0 ~ 1.5
- 能力值 1.0 → ZPD 范围：0.5 ~ 2.0

### 4. 难度值与能力值的关系

**相同尺度**：难度值和能力值使用相同的尺度（-3.0 到 3.0）

**IRT 模型中的关系**：
- 当难度值 = 能力值时，答对概率约 50%
- 当难度值 < 能力值时，答对概率 > 50%（题目较容易）
- 当难度值 > 能力值时，答对概率 < 50%（题目较难）

**实际意义**：
- 难度值 -1.0：基础题，大多数学生能答对
- 难度值 0.0：中等题，中等水平学生有 50% 概率答对
- 难度值 1.0：较难题，需要较高能力才能答对

## 💡 是否需要简化？

### 当前设计的优缺点

**优点**：
1. ✅ **符合 IRT 标准**：能力值和难度值使用相同尺度是 IRT 模型的标准做法
2. ✅ **科学准确**：基于教育测量学理论，换算关系有理论依据
3. ✅ **灵活扩展**：可以方便地添加新指标（如百分位排名）

**缺点**：
1. ❌ **不够直观**：能力值 -3 到 3 对普通用户不够友好
2. ❌ **年级等值复杂**：分段函数可能让用户困惑
3. ❌ **多个指标**：用户可能不清楚各指标的含义和关系

### 简化建议

#### 方案 1：保持现状，加强说明（推荐）

**理由**：
- IRT 模型使用 -3 到 3 是行业标准
- 简化可能损失精度和科学性
- 通过更好的说明和可视化可以解决理解问题

**改进措施**：
1. 在结果页添加换算关系说明
2. 提供换算工具/表格
3. 用更直观的语言解释各指标

#### 方案 2：简化显示，保留完整数据

**做法**：
- 主要显示标准分数和年级等值（更直观）
- 能力值和难度值作为技术指标，在详细报告中显示
- 提供"显示技术细节"选项

#### 方案 3：统一为单一指标

**不推荐**：会损失 IRT 模型的优势和多维度信息

## 📋 建议的改进

### 1. 添加换算关系说明

在结果页面添加一个"分数说明"部分，解释各指标的含义和关系。

### 2. 提供换算工具

创建一个简单的换算工具，用户可以输入能力值，查看对应的标准分数和年级等值。

### 3. 优化显示方式

- 主要显示：标准分数、年级等值（用户最关心的）
- 次要显示：能力值、ZPD（技术细节）
- 隐藏：难度值（除非用户需要）

### 4. 添加可视化

用图表展示：
- 能力值分布图
- 标准分数与年级等值的对应关系
- ZPD 范围的可视化

## 🎯 推荐方案

**建议采用方案 1 + 部分方案 2**：

1. **保持 IRT 标准**：不改变能力值和难度值的尺度
2. **优化显示**：
   - 主要显示标准分数和年级等值
   - 能力值作为次要信息
   - 添加"查看详细说明"链接
3. **加强说明**：
   - 在结果页添加换算关系说明
   - 提供换算示例
   - 解释各指标的实际意义

这样既保持了科学性，又提升了用户体验。
