# 功能更新说明

## 新增功能

系统已添加以下 STAR/AR 系统借鉴的功能：

### 1. 标准化评分系统
- **标准分数（Scaled Score）**：0-1000 分制
- **年级等值（Grade Equivalent, GE）**：显示相当于哪个年级的水平

### 2. 最近发展区（ZPD）
- 推荐题目难度范围
- 帮助选择适合的练习题

### 3. 预测性分析
- 预测 3 个月后的能力值
- 显示预期成长速度和置信度

### 4. 技能诊断
- 知识点子技能分析
- 错误类型识别

### 5. 测试频率建议
- 根据能力水平推荐测试频率
- 成长追踪建议

### 6. 推荐学习目标
- 短期目标（1-2个月）
- 长期目标（3-6个月）

## 如何查看新功能

### 方法 1：重启服务（推荐）

1. **重启后端服务**：
   ```bash
   cd math-test
   # 停止当前后端服务（Ctrl+C）
   ./start_backend.sh
   ```

2. **重启前端服务**：
   ```bash
   # 停止当前前端服务（Ctrl+C）
   ./start_frontend.sh
   ```

3. **清除浏览器缓存**：
   - 按 `Ctrl+Shift+R` (Windows/Linux) 或 `Cmd+Shift+R` (Mac) 强制刷新
   - 或打开开发者工具，右键刷新按钮选择"清空缓存并硬性重新加载"

### 方法 2：完成一次新测试

1. 访问测试页面
2. 完成一次新的测试（15-20题）
3. 查看结果页面，应该能看到所有新功能

### 方法 3：检查 API 响应

打开浏览器开发者工具（F12），在 Network 标签中查看 `/api/test/<session_id>/result` 的响应，应该包含以下字段：
- `scaled_score`
- `grade_equivalent`
- `zpd`
- `prediction`
- `skill_diagnosis`
- `testing_frequency`
- `goals`

## 故障排查

如果仍然看不到新功能：

1. **检查后端日志**：确认后端服务正在运行新代码
2. **检查前端控制台**：查看是否有 JavaScript 错误
3. **检查 API 响应**：确认后端返回了所有新字段
4. **清除浏览器缓存**：确保加载的是最新代码

## 测试建议

完成一次测试后，结果页面应该显示：
1. ✅ 标准分数和年级等值（在顶部统计区域）
2. ✅ ZPD 最近发展区（推荐题目难度范围）
3. ✅ 能力预测（3个月后预测）
4. ✅ 推荐学习目标（短期和长期）
5. ✅ 测试频率建议（包含成长追踪提示）
6. ✅ 技能诊断（错误类型和子技能分析）
